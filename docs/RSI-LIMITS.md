# これで「自律的な再帰的自己改善（RSI）」は実現できるか？

**短い答え: いいえ。本ツールは RSI の“実現装置”ではなく“封じ込め装置”です。**
むしろ「それは内部だけでは実現できない」ことを、実コードで示す側に立っています。

> RSI（recursive self-improvement）= システムが自分自身を改善し、改善が再帰的に複利化する
> （改善された系がさらに改善上手になる）こと。危険／本質的なのは、系が**自分の改善機構そのもの
> （最適化器・評価器・メタ層）**を書き換えていく版です。

このツールが何かは [EXPLAINER.md](EXPLAINER.md)、使い方は [USAGE.md](USAGE.md) を参照。

---

## 1. 構造的に — そもそも自律ループになっていない（意図的に）

- 候補は**外部供給**（`BuilderDir` = 人間／別途用意したファイル）。LLM builder は配線だけで
  この workflow では自動起動しない（claude-in-claude 回避）。
- 採否の gate は**凍結された決定的ロジック**（人間が書いた）で、自己改変しない。
- 改善対象は**標的の関数**であって、**自分の改善機構ではない**。
- DESIGN の **Ring2 = 人間ゲート**が残る。

「自律化する」= この人間ゲートを外す = **後述 R2 の失敗に対する唯一の床を外す**こと。

---

## 2. 原理的に — 内部測定には天井がある（このツールで実証済み）

RSI の核心は「最適化器が**自分の評価器（スコアボード）**を最適化し始める」点。内部の測定・検査で
縛れるのは **fast / gross / in-band**（速い・粗い・観測済み次元）の失敗だけで、次の3つは内部で閉じません。

1. **検出器と脅威の「粒度ズレ」**
   実改善ゼロの候補が**計測を捏造して ADOPT に到達**するのを実コードで示しました
   （[`../skeleton/test_security.py`](../skeleton/test_security.py)）。「検出器が緑でも脅威は通る」。
   in-process の防御は**境界ではない**（`os._exit` 経路で依然 forge 可能）。自律 RSI なら
   最適化器はこの穴を必ず突きます。

2. **メタ監視層の「適応 vs 静的」二択**
   評価器を凍結すれば、進化する最適化器に追い抜かれる。評価器を進化させれば、**自己参照腐敗
   （＝RSI そのもの）**。逃げ場のない二択で、これが RSI を内部で安全化できない本質です。

3. **construct validity（KPI vs 現実）**
   KPI は緑のまま、KPI が代理する**現実そのもの**を有害化できる（依存・ダークパターン・externality）。
   測定値は緑、悪いのは測っている世界。内部測定では原理的に不可視。

さらに、確証には毎回**新しいデータ**が要りますが、再帰的に回すほど holdout は枯渇し、
**「本当に新しいデータ」は内部では作れません**。本ツールの query-budget は、これを
「黙って過学習」でなく**「枯渇で停止」に変える**装置です。**止まるのが正直な終点**であって、
無限再帰の実現ではありません。

---

## 3. では何ができるのか（正直な肯定）

- **外部／人間が用意した改善候補を、厳密に採否する「決定の床」。** 「速くなったつもり」を一通り
  潰す（計測ノイズ・多重比較・winner's curse・テスト範囲外の欠陥・計測捏造）。
- 仮に自動改善を回すとしても、その **R1（速い・粗い・観測済み次元）の guardrail** を提供する
  ＝**必要条件であって十分条件ではない**。
- 天井を**可視化・loud 化**する（holdout 枯渇で停止／整合性保証を撤回し境界を明記）。

→ つまり RSI を**「実現」ではなく「封じ込める」**。DESIGN も明言しています:
**「安全な RSI は未解決の研究課題。本設計は封じ込めるのであって解決しない」**
（[`../DESIGN.md`](../DESIGN.md) の RSI 節 / [`../DESIGN-SUPPLEMENT-v2.md`](../DESIGN-SUPPLEMENT-v2.md)）。

---

## 4. 自律 RSI に本当に要るもの（＝このツールの外側）

- **異種モデル族**（同族 Claude×Claude は盲点を共有する、と検証の caveat にも明記）
- **人間の価値判断** / **第三者・規制監査** / **新規データの外部供給** / 実行の **OS 分離**

これらは**内部化した瞬間に腐ります**（§2-2 の二択に戻る）。だから「**外部アカウンタビリティでしか
床にならない**」。メタ層の RSI 自体は、依然として**未解決の研究問題**です。

## まとめ

| 問い | 答え |
|---|---|
| これで自律的・再帰的な自己改善を実現できる？ | **いいえ** |
| では何が実現できる？ | **人間／外部主導の、厳密に検証された改善の“採否の床”** |
| RSI に対する本ツールの役割は？ | **実現ではなく封じ込め**（R1 を縛り、天井を loud にする） |
| 残る天井は？ | 計測捏造の真の防御・construct drift・新規データ・メタ層 RSI ＝**すべて外部** |

本プロジェクトの価値は、この境界を**塞いだふりをせず、実コード（動く exploit）で示した**ことにあります。
